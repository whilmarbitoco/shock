Why Shock?

🚀

No Servlet Container

Runs on a raw ServerSocket. No Tomcat, no Jetty, no Spring Boot. Starts in milliseconds.

🗄️

Active Record ORM

HasMany, BelongsTo relations, eager loading, migrations — all built in. No Hibernate needed.

🔀

Dynamic Routing

Clean route definitions with middleware support. GET, POST, PUT, DELETE, PATCH — all first-class.

🧩

Middleware Pipeline

Global and per-route middleware. Auth, logging, sessions, CORS — plug them in where you need them.

📐

Template Engine

{{variable}}, {% if %}, {% for %}, {% else %} — simple, fast, no external deps.

🔌

Multi-Database

MySQL, PostgreSQL, SQLite via dialect abstraction. Maven profiles keep your POM clean.

🏊

Built-In Connection Pool

Lightweight ArrayBlockingQueue pool. No HikariCP dependency needed.

🐳

Docker-Ready

Multi-stage Dockerfile, docker-compose with DB profiles. Production-ready out of the box.

Quick Start

1

Add the dependency

<dependency>
    <groupId>org.whilmarbitoco</groupId>
    <artifactId>mvc-framework</artifactId>
    <version>1.0-SNAPSHOT</version>
</dependency>
2

Define your routes

public class Routes extends RouteRegistry {
    public void register() {
        router.get("/", HomeController.class, "index");
        router.get("/users", UserController.class, "list");
        router.post("/users", UserController.class, "create");
    }
}
3

Write a controller

public class HomeController {

    public void index(Request req, Response res) {
        res.view("index", Map.of(
            "title", "Welcome to Shock"
        ));
    }
}
4

Write a template

<h1>{{title}}</h1>

{% if users.empty %}
    <p>No users yet.</p>
{% else %}
    <ul>
    {% for user in users %}
        <li>{{user.name}} — {{user.email}}</li>
    {% endfor %}
    </ul>
{% endif %}
5

Run it

# Build
mvn package -B -P sqlite

# Run
java -cp "target/*:." com.app.Main

# Or with Docker
docker compose up --build
